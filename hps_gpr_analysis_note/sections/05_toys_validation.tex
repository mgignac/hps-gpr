\section{Toy Generation and Validation Studies}
\label{sec:toys}

\subsection{Scope of the current validation campaign}

Unless a figure caption states otherwise, the injection/extraction studies summarized
in this note correspond to the 95\% confidence-level workflow with blind width
$1.64\,\sigma_m$ and \num{10000} toys per mass-strength point. The completed summary
suite currently covers 2015, 2016 10\%, and the 2015+2016 combination. The combined
search range will ultimately be quoted only in the physics-relevant interval
\SIrange{35}{130}{MeV}; odd behavior outside that interval is therefore noted only
briefly and not overinterpreted.

\subsection{Three toy-generation modes}

The current validation program uses three distinct toy constructions.

\subsubsection{Analytic functional-form pseudoexperiments}

The first toy category uses an explicit analytic fit to the smooth background spectrum
and then fluctuates pseudo-data around that fit. The repository already contains a

gross spectral shape independently of the GP formalism. They are also the most direct
way to test whether a smooth functional description already captures the dominant
background behavior in a given subset.

\subsubsection{Conditional blind-window GP toys}

The default fast toy mode in the package keeps the sideband-trained GP fixed. At each
mass point the procedure is:
\begin{enumerate}
\item fit the GP on sidebands only and obtain the blind-window mean vector $\bm{b}$ and
covariance $\bm{C}$;
\item draw a non-negative multivariate-normal background realization from
$\mathcal{N}(\bm{b},\bm{C})$;
\item inject signal in the blinded region;
\item Poisson-sample the observed blind-window counts.
\end{enumerate}
This mode tests the extraction machinery under the GP posterior implied by the real
data, without rebuilding the global pseudo-dataset each time.

\subsubsection{Full procedural GP-mean/refit pseudoexperiments}

The slower but more realistic mode generates a full histogram-level pseudoexperiment
from the GP global-fit mean, injects signal either over the full template or only in
the blind window, and then refits the GP on toy sidebands before extraction. In the
package this corresponds to \texttt{inj\_refit\_gp\_on\_toy = true}. This mode is the
closest match to the later \texttt{v15\_8} notebook procedure and is the appropriate
tool for assessing how much the full GP refit can absorb injected signal.

\subsection{Asimov versus Poisson reference construction}

When injections are parameterized in units of $\sigma_A$, the package first defines a
reference extraction uncertainty $\sigma_{A,\mathrm{ref}}(m)$. Two reference modes are
implemented:
\begin{itemize}
\item \textbf{Asimov mode:} set the reference counts equal to the predicted background
mean, $n_i^{\mathrm{ref}} = b_i$. This isolates the deterministic sensitivity implied
by the fitted background model.
\item \textbf{Poisson mode:} Poisson-fluctuate the reference counts before extracting
$\sigma_{A,\mathrm{ref}}$. This embeds counting noise already at the normalization
stage.
\end{itemize}
The completed 95\% CL studies summarized here use the Asimov reference convention,
matching the current study configs and extraction-display setups.

\subsection{Poisson versus multinomial signal injection}

Signal injection itself is treated separately from the $\sigma_A$ reference
construction. The package supports:
\begin{itemize}
\item \textbf{Poisson-total injection}, in which each bin receives a Poisson draw with
mean $A_{\mathrm{inj}}w_i$ and the total injected signal fluctuates from toy to toy;
\item \textbf{multinomial fixed-total injection}, in which the total signal count is
fixed and only the distribution across bins fluctuates according to the Gaussian
template weights.
\end{itemize}
The blind-window injection studies that populate the repository-level summary suite use
\texttt{inj\_mode = poisson}. The extraction-display figures, by contrast, use
\texttt{multinomial} injection together with a full toy GP refit, so they visualize a
fixed requested signal strength more cleanly.

Only the Poisson-total injection mode has a genuine horizontal spread in the x-variable
of linearity or bias plots, because the realized total injected count fluctuates. When
multinomial injection is used, the true horizontal uncertainty is zero, and any x-axis
whiskers would be merely a visual reference rather than an uncertainty carried by the
toys.

\subsection{Injection summary suite}

The \texttt{injection\_summary\_9} directory contains the publication-style summaries
generated from the completed 2015, 2016, and combined 95\% CL toy campaign. The
primary diagnostic classes are:
\begin{itemize}
\item \textbf{Linearity} plots, which test whether $\langle\hat A\rangle$ follows
$A_{\mathrm{inj}}$;
\item \textbf{Bias} plots, which track $\langle\hat A\rangle-A_{\mathrm{inj}}$;
\item \textbf{Pull-width} plots, which test whether
$\sigma[(\hat A-A_{\mathrm{inj}})/\sigma_A]$ remains close to unity;
\item \textbf{Coverage} plots, which report the empirical fractions inside the nominal
$1\sigma$ and $2\sigma$ pull bands;
\item \textbf{Heatmaps} of pull mean and pull width over mass and injection strength;
\item \textbf{$Z$-calibration residuals}, which compare toy-extracted significance to
the injected significance scale;
\item \textbf{combined-search-power} studies, which visualize the gain from combining
independent datasets.
\end{itemize}

\subsection{Combined search-power studies}

The combined-search-power plots are intended to show why the
2015+2016 combination improves over either constituent dataset alone. The
\texttt{combined\_search\_power\_scenarios} plot compares scenario-level combined
significances, while the constituent $5\sigma$ requirement plot translates a target
combined excess into the local $Z$ and $p_0$ that each dataset would need to provide.
The allocation plots at \SIlist{40;80;115}{MeV} then show how the required signal is
distributed between 2015 and 2016 according to the inverse-variance weighting encoded
by $\sigma_{A,\mathrm{ref}}$.
