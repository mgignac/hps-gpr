\section{Toy Generation and Validation Studies}
\label{sec:toys}

\subsection{Scope of the current validation campaign}

Unless a figure caption states otherwise, the injection/extraction studies summarized
in this note correspond to the 95\% confidence-level workflow with blind width
$1.64\,\sigma_m$ and \num{10000} toys per mass-strength point. The completed summary
suite presently covers 2015, 2016 10\%, and the 2015+2016 shared-$\eps^2$
combination. The physics-facing interpretation of the combined search will ultimately
be restricted to the mass interval \SIrange{35}{130}{MeV}; behavior outside that range
is retained for diagnostics but is not emphasized in the discussion below.

The copied toy-validation, combined-search, and upper-limit figures bundled with this
note were produced before the final adoption of the leakage-retaining signal-template
convention described in Section~\ref{sec:methodology}. They remain useful because they
already expose the main qualitative features of the extraction and combination
machinery, but they should be interpreted as provisional until the corresponding rerun
is completed with the full-range template definition.

\subsection{Three toy-generation modes}

The current validation program uses three distinct pseudoexperiment constructions, each
answering a different statistical question.

\subsubsection{Analytic functional-form pseudoexperiments}

The first toy category starts from an explicit analytic fit to the smooth mass spectrum
and fluctuates pseudo-data around that closed-form background model. This approach is
especially useful for the earliest closure tests because it isolates the question of
gross spectral smoothness from the more complicated question of GP conditioning. The
repository already contains a representative 2021 example showing that the selected
analytic form tracks the 0.03\% subset well enough to seed this kind of study. The
corresponding 2015 and 2016 validation plots have not yet been exported and are left
as explicit placeholders in the note structure.

\subsubsection{Conditional blind-window GP toys}

The fastest production toy mode keeps the sideband-trained GP fixed at each test mass.
Operationally, the procedure is to fit the GP on sidebands only, translate the
prediction into a blinded-bin mean vector $\bm{b}$ and covariance $\bm{C}$, draw a
background realization from the implied multivariate distribution, add signal in the
blinded bins, and finally Poisson-sample the observed counts. Because the sideband fit
is not rebuilt for every toy, this mode is ideal for stress-testing the extraction
machinery under the posterior background model already implied by the real data. It is
also the most efficient way to generate large expected-band ensembles.

\subsubsection{Full procedural GP-mean/refit pseudoexperiments}

The more realistic but slower toy mode constructs a new full-histogram pseudoexperiment
from the GP-derived global background expectation, injects signal using the
full-histogram Gaussian template, and then reruns the sideband-to-blind prediction on
the toy before signal extraction. In the current package this corresponds to enabling
\texttt{inj\_refit\_gp\_on\_toy}. This is the mode that most directly probes whether the
full analysis chain absorbs part of the injected signal through GP refitting or through
the profiled nuisance model. It is therefore the decisive validation mode for the
remaining signal-absorption question.

\subsection{Asimov versus Poisson reference construction}

When injections are specified in units of $\sigma_A$, the package first defines a
reference uncertainty $\sigma_{A,\mathrm{ref}}(m)$. Two conventions are implemented.
In \textbf{Asimov mode}, the reference counts are set equal to the predicted background
mean, $n_i^{\mathrm{ref}}=b_i$, so the normalization reflects the deterministic local
sensitivity of the fitted model. In \textbf{Poisson mode}, the reference counts are
Poisson-fluctuated before $\sigma_{A,\mathrm{ref}}$ is extracted, so counting noise is
already present at the normalization stage. The completed 95\% CL studies summarized
here use the Asimov convention, which matches the current study configurations and is
better suited to comparing different toy constructions on the same footing.

\subsection{Poisson versus multinomial signal injection}

Signal injection is handled separately from the choice of $\sigma_A$ reference.
\textbf{Poisson-total injection} assigns each bin an independent Poisson draw with mean
$A_{\mathrm{inj}}w_i$, so the realized total injected signal fluctuates from toy to toy.
\textbf{Multinomial fixed-total injection} keeps the total injected yield fixed and
distributes it across bins according to the Gaussian template weights. In the current
implementation $A_{\mathrm{inj}}$ denotes the total local Gaussian yield, while the
expected signal appearing inside the blinded bins is
$f_{\mathrm{blind}}A_{\mathrm{inj}}$ with $f_{\mathrm{blind}}<1$ for the nominal
$1.64\,\sigma_m$ window.

This distinction explains the horizontal error bars that appear in some injection
summary plots. Only Poisson-total injection produces a genuine spread in the realized
horizontal coordinate because the total injected signal itself fluctuates. For
multinomial fixed-total injection the x-axis value is exact, so any horizontal whisker
would merely be a plotting convention rather than a toy-derived uncertainty. The
blind-window injection studies that populate the summary suite use
\texttt{inj\_mode = poisson}, whereas the extraction-display plots prefer a fixed-total
injection so that the visual comparison between requested and fitted signal strength is
cleaner.

model is doing at the level of individual mass hypotheses rather than only through

unconstrained closure estimator is allowed to fluctuate below zero. The 2015
\SI{30}{MeV} display provides a lower-mass comparison with a similarly transparent

\subsection{Injection summary suite}

The \texttt{injection\_summary\_9} suite is the main summary product of the completed
2015, 2016, and combined toy campaign. The linearity panels test whether the mean
extracted amplitude tracks the injected amplitude. The bias panels display the same
information after subtracting the truth, so persistent offsets are immediately visible.
The pull-width and coverage summaries then test whether the reported extraction
uncertainty is calibrated correctly. Finally, the heatmaps and $Z$-calibration figures
localize where mass dependence or signal strength dependence causes the extraction to
depart from ideal behavior. Because these figures predate the final leakage-retaining
rerun, they should be read as baseline diagnostics rather than as frozen final
performance statements.

\subsection{Combined search-power studies}

The combined-search-power figures explain qualitatively why the shared-$\eps^2$
combination can outperform either constituent dataset alone. The scenario-level summary
shows how the combined local significance grows under different assumptions about the
relative strength of the contributing datasets. The constituent-requirement plot
recasts a target combined $5\sigma$ excess into the local $Z$ and $p_0$ each dataset
would need to supply. The mass-specific allocation plots at \SIlist{40;80;115}{MeV}
then visualize how the required signal partitions between 2015 and 2016 according to
the inverse-variance weighting implicit in $\sigma_{A,\mathrm{ref}}$. These figures are
best understood as interpretation aids for the shared-likelihood machinery rather than
as direct limit-setting inputs.
